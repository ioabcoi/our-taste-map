# 기능 목록 (Feature List)

> 최종 수정일: 2026.02.12

## 📋 개요
Our-Taste-Map(맛멋집)의 기능을 정의하고 우선순위를 관리하는 문서입니다.

**프로젝트 방향:**
- v1.0: 우리 부부 전용 지도 (로그인 없음, 단일 계정)
- v2.0: 다중 사용자 지원 (회원가입/로그인 추가)

---

## 🎯 v1.0 MVP (Minimum Viable Product) - 우리 부부 전용

### 핵심 컨셉
- 로그인 없이 바로 사용
- 우리 둘만 데이터 공유
- 읽기 전용 공유 링크로 친구들에게 구경시키기

---

### 1. 지도 뷰 📍
**우선순위: 🔴 High**

- [ ] 카카오맵 API 연동
- [ ] 기본 지도 표시 (확대/축소/이동)
- [ ] 현재 위치 표시
- [ ] 커스텀 마커 표시
  - 맛집 마커 (🍽️)
  - 카페 마커 (☕)
  - 멋진 장소 마커 (✨)
- [ ] 마커 클릭 시 간단한 정보 미리보기 (이름, 별점)
- [ ] 마커 더블클릭 시 상세 모달 열기

**기술적 고려사항:**
- 지도 API: 카카오맵 vs 네이버맵 vs 구글맵
- 마커 커스터마이징 방법
- 성능: 마커가 많을 때 클러스터링 필요한지?

**v1.0 단순화:**
- 권한 체크 불필요 (우리만 씀)
- 복잡한 공유 기능 제외

---

### 2. 리스트 뷰 📝
**우선순위: 🔴 High**

- [ ] 지도/리스트 뷰 토글 버튼
- [ ] 장소 목록 카드 형태로 표시
  - 대표 사진 썸네일
  - 이름, 카테고리
  - 별점, 한줄평
  - 주소 (또는 지역)
  - **방문일** ⭐ NEW
- [ ] 리스트에서 장소 클릭 시 상세 모달 열림
- [ ] 최신순 / 별점순 / 이름순 / **방문일순** 정렬 ⭐ NEW

**UI/UX 고려사항:**
- 카드 레이아웃: 1열? 2열? (모바일/PC)
- 무한 스크롤 vs 페이지네이션

---

### 3. 카테고리 필터링 🏷️
**우선순위: 🔴 High**

- [ ] 전체 보기
- [ ] 맛집만 보기
- [ ] 카페만 보기
- [ ] 멋진 장소만 보기
- [ ] 필터 적용 시 지도 + 리스트 동시 업데이트

**추가 고려사항:**
- 다중 선택 가능? (맛집 + 카페 동시에)
- 필터 UI: 탭? 드롭다운? 체크박스?

---

### 4. 장소 등록/수정/삭제 ✏️
**우선순위: 🔴 High**

#### 등록
- [ ] 장소 검색 (지도 API 활용)
- [ ] 직접 입력 옵션
- [ ] 필수 정보 입력
  - 장소명
  - 주소
  - 카테고리 선택 (맛집/카페/멋진 장소)
  - 별점 (⭐ ~ ⭐⭐⭐⭐⭐)
- [ ] 선택 정보 입력
  - **방문일 (기본값: 오늘 날짜)** ⭐ NEW
  - 한줄평 (최대 100자)
  - 상세 리뷰 (자유 형식)
  - 추천 메뉴
  - 사진 업로드 (최대 10장)

#### 수정/삭제
- [ ] 장소 정보 수정
- [ ] 장소 삭제 (확인 알림)
- [ ] 사진 추가/삭제/순서 변경

**기술적 고려사항:**
- 이미지 저장소: Firebase Storage vs Cloudinary
- 이미지 최적화 (리사이징, 압축)
- 사진 업로드 개수 제한: 최대 10장
- 사진 1장당 용량 제한: 5MB

**v1.0 단순화:**
- 권한 체크 불필요 (우리만 등록/수정/삭제)
- 수정 이력 추적 불필요

### 5. 별점 및 리뷰 ⭐
**우선순위: 🔴 High**

- [ ] 별점 입력 (1~5점, 0.5점 단위)
- [ ] 한줄평 입력 (최대 100자)
- [ ] 상세 리뷰 입력 (자유 형식, 선택사항)
- [ ] 추천 메뉴 입력 (자유 형식)
- [ ] **방문일 입력 (수동, 선택사항)** ⭐ NEW
  - 빈칸이면 오늘 날짜 자동 입력
  - 과거 날짜 선택 가능 (달력 UI)
  - 미래 날짜는 선택 불가
- [ ] 작성일 자동 기록
- [ ] 수정일 자동 기록

**추가 설명:**
- **방문일** (`visitDate`): 실제 방문한 날짜 (사용자 입력, 선택사항)
  - 예: "2024년 8월에 갔었는데 12월에 기록" → 방문일: 2024.08.15
- **작성일** (`createdAt`): 데이터를 처음 등록한 날짜 (자동)
- **수정일** (`updatedAt`): 마지막으로 수정한 날짜 (자동)

## 💭 기술 스택 고려사항 (v1.0 최적화)

### 단순화 포인트
```javascript
// ✅ v1.0: 단순한 구조
const db = firebase.firestore()
db.collection('places').get()

// ⏳ v2.0: 사용자별 분리
db.collection('places')
  .where('userId', '==', currentUser.uid)
  .get()
```

### Database 구조 (확장 가능하게)
```javascript
places: {
  placeId: {
    name: "맛집이름",

    category: "restaurant", // restaurant, cafe, spot
    rating: 4.5,
    shortReview: "한줄평",
    detailReview: "상세리뷰",
    recommendMenu: "추천메뉴",
    photos: ["url1", "url2"],
    groups: ["인천", "동탄"], // 즐겨찾기 그룹
    latitude: 37.xxx,
    longitude: 127.xxx,
    visitDate: "2024-08-15", // ⭐ NEW - 방문일
    createdAt: timestamp,    // 작성일
    updatedAt: timestamp     // 수정일
    // userId: "user_id" // v2.0에서 추가할 필드
  }

groups: {
  groupId: {
    name: "인천",
    color: "#FF5733", // 그룹별 색상

  }

```

## 💬 논의 필요 사항

1. **카테고리 세분화 필요한가?**
   - v1.0: 맛집/카페/멋진 장소 (3개만)
   - v1.5: 세부 카테고리 추가? (한식/양식/브런치카페 등)

2. **사진 용량/개수 제한**
   - 제안: 장소당 최대 10장, 1장당 5MB
   - Firebase Storage 무료 용량: 5GB

3. **지도 API 선택**
   - 카카오맵: 국내 정확도 높음, 무료 할당량 괜찮음
   - 네이버맵: 카카오랑 비슷
   - 구글맵: 글로벌, 유료 부담

4. **배포 환경**
   - 제안: Vercel (무료, 자동 배포)
   - 도메인: v1.0은 불필요, v2.0부터 구매

5. **데이터 접근 방법**
   - v1.0: 환경변수에 Firebase Config 저장
   - 보안: 읽기 전용 공유 링크만 외부 노출

6. **방문일 입력 방식** ⭐ NEW
   - 달력 UI vs 직접 입력
   - 기본값: 오늘 날짜 자동 입력
   - 빈칸 허용? (작성일로 대체)
