# 사용자 페르소나 (User Persona)

> 최종 수정일: 2026.02.11

## 📋 개요
Our-Taste-Map의 사용자를 정의하고, 그들의 니즈와 사용 맥락을 이해하기 위한 문서입니다.

**프로젝트 방향:**
- v1.0: 우리 부부 전용 (Primary Users)
- v2.0: 일반 사용자 확장 (Secondary Users)

---

## 👥 v1.0 Primary Users (주 사용자)

### 페르소나 1: 우리 부부

**기본 정보**
- **관계**: 부부 (또는 커플)
- **나이**: 20대 후반 ~ 30대 초반
- **직업**: 직장인
- **거주지**: 수도권 (인천, 수원, 동탄 등 여러 지역 이동)
- **기술 숙련도**: 중상 (웹/앱 개발 가능, 토이프로젝트 진행 중)
- **취미**: 주말마다 맛집/카페 탐방

**특징**
- 🍽️ 주말마다 새로운 맛집, 카페 방문
- 📸 SNS에서 본 곳을 자주 가봄
- 💑 함께 다녀온 곳을 기록하고 추억하고 싶음
- 🗺️ "여기 언제 왔었지? 뭐가 맛있었더라?" 자주 발생

**Pain Points (불편한 점)**

**1. 기존 맛집 앱의 문제**
- 📱 타인의 리뷰가 너무 많아 내가 간 곳 찾기 어려움
  - "우리가 간 맛집만 모아보고 싶은데..."
  - 망고플레이트, 카카오맵은 타인 리뷰 위주
- 💰 광고성 리뷰, 조작된 별점 때문에 신뢰도 낮음
- 🎯 내 취향과 맞지 않는 추천 알고리즘

**2. 기록의 어려움**
- 📍 카카오맵 즐겨찾기는 너무 단순함
  - 별점, 사진, 메모 제대로 못 남김
  - 언제 갔는지 기억 안 남
- 📷 인스타그램에 올린 맛집 사진
  - 나중에 찾기 힘듦
  - "여기 어디였더라?" 검색 어려움
- 📝 메모장에 적어둔 맛집 리스트
  - 관리가 안 됨
  - 사진이랑 연결 안 됨

**3. 지역별 관리 불편**
- 🗺️ "인천 갈 때 가볼 곳" 따로 보고 싶은데 섞여있음
- 📱 여러 앱 왔다갔다 해야 함
- 🔍 검색해도 내가 간 곳인지 헷갈림

**Goals (목표)**

**단기 목표 (v1.0)**
- ✅ 우리가 다녀온 곳만 모아서 보고 싶다
- ✅ 별점, 사진, 한줄평을 함께 기록하고 싶다
- ✅ 지역별로 구분해서 관리하고 싶다 (인천, 동탄, 수원...)
- ✅ 나중에 재방문할 때 "여기 뭐가 맛있었지?" 바로 확인
- ✅ 지도에서 한눈에 우리가 다녀온 곳 보기

**장기 목표 (v2.0)**
- 🎯 친구들에게 "우리 맛집 지도" 자랑하기
- 🎯 친구도 자기 지도 만들 수 있게 하기
- 🎯 토이프로젝트를 실제 서비스로 확장

## 🎬 사용 시나리오 (v1.0 기준)

### 시나리오 1: 주말 맛집 탐방 후 기록 📝

**상황:**
```
토요일 오후, 동탄 신도시 새로 오픈한 파스타집 방문
→ 크림 파스타 주문, 너무 맛있어서 사진 3장 찍음
→ "30분 웨이팅 했지만 그만한 가치 있었어!"
→ 집에 돌아옴
```

**Our-Taste-Map 사용:**
```
1. PC/모바일에서 Our-Taste-Map 접속 (로그인 불필요)
2. 지도에서 "동탄" 지역 찾기
3. "+ 장소 추가" 버튼 클릭
4. "파스타 이야기" 검색 (카카오맵 API로 자동 주소 입력)
5. 정보 입력:
   - 카테고리: 맛집
   - 별점: ⭐⭐⭐⭐⭐ (5.0)
   - 한줄평: "크림 파스타 존맛! 웨이팅 30분 각오"
   - 추천 메뉴: "크림 파스타, 알리오올리오"
   - 사진 3장 업로드
   - 즐겨찾기 그룹: "동탄" 선택
6. 저장 → 지도에 🍽️ 마커 추가됨
```

**결과:**
- ✅ 다음에 동탄 가면 바로 찾을 수 있음
- ✅ 사진 보면서 "여기 진짜 맛있었지" 회상 가능
- ✅ 친구가 "동탄 맛집 추천해줘" 하면 바로 보여주기

---

### 시나리오 2: 다음 주말 맛집 찾기 🔍

**상황:**
```
금요일 저녁, 회의 중
아내: "주말에 인천 놀러갈까?"
남편: "좋지! 인천에 우리 가본 곳 있나?"
```

**Our-Taste-Map 사용:**
```
1. 스마트폰으로 Our-Taste-Map 접속
2. 왼쪽 메뉴에서 "인천" 그룹 선택
3. 지도에 인천 지역 맛집/카페 마커만 표시됨
4. "아! 차이나타운에 짬뽕집 갔었지!"
5. 마커 클릭 → 별점 4.5, "짬뽕 진하고 맛있음" 확인
6. "여기 또 가자!"
```

**결과:**
- ✅ 검색 필요 없이 바로 확인
- ✅ 과거 경험 기반 선택 가능
- ✅ 시간 절약, 만족도 UP

---

### 시나리오 3: 친구에게 맛집 추천 💬

**상황:**
```
카카오톡 단톡방
친구: "동탄에 맛있는 카페 있어?"
```

**Our-Taste-Map 사용:**
```
1. Our-Taste-Map에서 "동탄" + "카페" 필터 적용
2. 별점 높은 순 정렬
3. "브라운스톤 카페" 선택
4. 상세 모달에서 사진, 리뷰 확인
5. 카톡 공유 버튼 클릭
   → "브라운스톤 카페 - 브런치 맛있어요! ⭐4.5"
   (v1.0: 수동 복붙 / v2.0: 링크 자동 생성)
```

**결과:**
- ✅ 신뢰도 높은 추천 (내가 직접 가본 곳)
- ✅ 사진으로 분위기 미리 알려줌

---

### 시나리오 4: "여기 언제 왔더라?" 확인 🤔

**상황:**
```
카페에서 대화 중
아내: "우리 여기 와본 적 있지 않아?"
남편: "어? 맞는데... 언제였지?"
```

**Our-Taste-Map 사용:**
```
1. 현재 위치 기준 주변 검색 (v1.0에서는 수동 검색)
2. 또는 카페 이름으로 검색
3. 마커 찾음 → 상세 모달 열기
4. "2023년 8월 작성" 확인
5. 사진 보면서 "아! 이때 아이스크림 먹었지!"
```

**결과:**
- ✅ 추억 회상
- ✅ 재방문 시 만족감 UP

## 👤 v2.0 Secondary Users (확장 사용자)

### 페르소나 2: 일반 사용자 (개인 맛집 관리자)

**기본 정보**
- **이름**: 수진 (가명)
- **나이**: 20대 중반 ~ 30대 초반
- **직업**: 다양 (직장인, 프리랜서, 학생 등)
- **특징**: 혼자 또는 친구들과 맛집 탐방
- **니즈**: 자기만의 맛집 지도 만들고 싶음

**v2.0에서 추가될 기능:**
- 회원가입/로그인
- 개인별 데이터 분리
- 내 지도 공개/비공개 설정
- 친구 초대 (선택)

---

### 페르소나 3: 공유 링크 방문자 (읽기 전용)

**기본 정보**
- **이름**: 친구들, 가족
- **목적**: 우리 부부가 공유한 맛집 구경
- **행동**: 읽기만 가능, 수정 불가

**v1.0 기능:**
- 지도 보기 (읽기 전용)
- 리스트 보기
- 상세 정보 확인

**v2.0 기능:**
- "나도 만들기" 버튼 → 회원가입 유도

## 🚫 Anti-Persona (타겟이 아닌 사용자)

### v1.0/v2.0 모두 해당 없음

**1. 실시간 트렌드 맛집을 찾는 사람**
- "지금 핫한 맛집" 찾는 사람
- 타인의 리뷰에 의존하는 사람
- → 망고플레이트, 카카오맵 리뷰가 더 적합

**2. 배달/포장 위주 사용자**
- 직접 방문보다 배달 위주
- 메뉴 정보, 영업시간이 더 중요
- → 배달앱이 더 적합

**3. 비즈니스 목적 사용자**
- 식당 사장님이 홍보 목적으로
- 맛집 인플루언서의 수익 목적
- → 우리 서비스 방향과 맞지 않음
