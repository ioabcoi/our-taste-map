# GitHub에서 메인 화면이 안 나오는 이유는...

```
🤔 현재 상황

GitHub 레포에 들어가면:

README.md는 보임
하지만 실제 웹사이트는 안 보임

왜냐하면:

지금은 코드만 올린 상태예요
웹사이트로 보려면 **배포(Deploy)**가 필요해요!

📦 GitHub vs 배포된 사이트

GitHub 레포지토리         배포된 웹사이트
(코드 저장소)             (실제 동작하는 사이트)
     ↓                          ↓
코드만 보임               실제 화면 보임
README.md 표시            React 앱 실행됨
```
→ 🚀 해결 방법: Vercel로 배포

## Vercel 자동 배포 (추천!)

### 1단계: Vercel 가입

```
https://vercel.com
→ GitHub 계정으로 로그인
```

### 2단계: Import Project

```
Vercel 대시보드
```

### 4단계: Deploy 버튼 클릭!

```
→ 1~2분 후 완성
→ URL 생성: our-taste-map.vercel.app
```

# 🚀 Vercel 배포 단계별 가이드

## 1단계: Vercel 가입

### 1. Vercel 사이트 접속

```
https://vercel.com
```

### 2. 로그인

```
우측 상단 "Sign Up" 또는 "Log In" 클릭
→ "Continue with GitHub" 선택
→ GitHub 계정으로 로그인
```

### 3. 권한 승인

```
Vercel이 GitHub 레포지토리 접근 요청
→ "Authorize Vercel" 클릭
```
✅ 가입 완료!

## 2단계: 프로젝트 Import

### 1. 대시보드에서

```
"Add New..." 버튼 클릭
→ "Project" 선택
```

### 2. GitHub 레포 연결

```
"Import Git Repository" 섹션에서
→ "our-taste-map" 레포지토리 찾기
→ "Import" 버튼 클릭
```

만약 레포가 안 보이면:
```
"Adjust GitHub App Permissions" 클릭
→ "our-taste-map" 레포지토리 체크
→ "Save" 클릭
```

## 3단계: 프로젝트 설정

Vercel이 자동으로 감지하지만, 확인해주세요:

```

Root Directory: ./
Build Command: npm run build
Output Directory: dist
Install Command: npm install
```

환경변수는 지금 Skip!

## 4단계: Deploy!

1. "Deploy" 버튼 클릭!

2. 기다리기 (1~2분)

```
Building... (빌드 중)
  ↓
Deploying... (배포 중)
  ↓
✓ Ready! (완료!)
```

3. 완성!

```
https://our-taste-map.vercel.app

또는

https://our-taste-map-[랜덤문자].vercel.app
```

## 🔧 해결 방법

### 1단계: 로컬에서 먼저 고치기

터미널 열고 프로젝트 폴더로:

```bash
cd our-taste-map
```

package.json 수정: (완료)

### 2단계: 로컬에서 테스트

```bash
# 기존 node_modules 삭제
rm -rf node_modules package-lock.json

# 또는 Windows
rmdir /s node_modules
del package-lock.json

# 다시 설치
npm install

# 빌드 테스트
npm run build
```

에러 없이 dist 폴더 생성되면 OK! ✅

### 3단계: Git에 푸시

```bash
git add package.json
git commit -m "fix: ESLint 의존성 충돌 해결"
git push
```

### 4단계: Vercel 자동 재배포

```
Git push하면 Vercel이 자동으로 다시 배포 시도!
→ 이번엔 성공할 거예요!
```
